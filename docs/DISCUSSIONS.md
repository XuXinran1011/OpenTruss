# GitHub Discussions 使用指南

本文档介绍如何使用 GitHub Discussions 参与 OpenTruss 项目的讨论。

## 什么是 GitHub Discussions？

GitHub Discussions 是一个论坛式的讨论空间，用于：

- 提出和讨论新功能想法
- 询问使用问题
- 分享经验和最佳实践
- 与社区成员交流

## 如何启用 Discussions

> **注意**: Discussions 功能需要在 GitHub 仓库设置中手动启用。

### 启用步骤

1. 访问仓库设置: https://github.com/XuXinran1011/OpenTruss/settings
2. 在左侧菜单中选择 "General" → "Features"
3. 找到 "Discussions" 部分
4. 勾选 "Discussions" 复选框
5. 点击 "Save changes"

## Discussions 分类

Discussions 通常分为以下几个分类（Categories）：

### 📢 Announcements（公告）

项目维护者发布的重要公告、版本更新、重大变更等。

- **用途**: 官方公告
- **发布权限**: 通常只有维护者可以发布

### 💡 Ideas（想法）

讨论新功能想法和产品改进建议。

- **用途**: 提出和讨论新功能
- **示例**: "建议添加批量导入功能"、"能否支持更多构件类型？"

### ❓ Q&A（问答）

询问使用问题，获取帮助。

- **用途**: 技术问题、使用问题
- **示例**: "如何配置 Memgraph？"、"Trace Mode 如何使用？"

### 💬 General（一般讨论）

一般性讨论、经验分享、最佳实践等。

- **用途**: 社区交流、经验分享
- **示例**: "使用 OpenTruss 的最佳实践"、"性能优化经验"

### 🎉 Show and tell（展示）

展示使用 OpenTruss 创建的项目、分享成果。

- **用途**: 项目展示、成果分享
- **示例**: "使用 OpenTruss 完成的第一个项目"

## 参与讨论

### 创建新讨论

1. 访问 Discussions 页面: https://github.com/XuXinran1011/OpenTruss/discussions
2. 点击 "New discussion" 按钮
3. 选择合适的分类
4. 填写标题和内容
5. 点击 "Start discussion"

### 回复讨论

1. 在讨论页面底部找到回复框
2. 输入回复内容
3. 可以使用 Markdown 格式
4. 点击 "Comment" 提交回复

### Markdown 支持

Discussions 支持完整的 Markdown 语法，包括：

- 代码块
- 图片
- 链接
- 列表
- 表格

## 讨论指南

### 创建好的讨论

一个好的讨论应该：

1. **清晰的标题**: 简洁明了地描述讨论主题
2. **详细的内容**: 提供足够的上下文信息
3. **正确的分类**: 选择最合适的分类
4. **相关标签**: 使用标签帮助分类和搜索

### 示例：好的讨论标题

✅ **好的标题**:
- "如何在生产环境中配置 Prometheus 监控？"
- "建议：支持批量导出多个检验批的 IFC"
- "Trace Mode 中拓扑修复的优化建议"

❌ **不好的标题**:
- "问题"
- "help"
- "bug???"

### 示例：好的讨论内容

```markdown
## 问题描述

我在使用 OpenTruss 时遇到了一个问题：
- 环境: Docker Compose
- 版本: 1.0.0
- 操作: 尝试导出检验批到 IFC

## 期望行为

应该能成功导出 IFC 文件

## 实际行为

导出的 IFC 文件无法在 Revit 中打开

## 相关代码/日志

```
[错误日志]
```

## 尝试的解决方案

1. 检查了 IFC 文件格式
2. 重新生成了检验批
```

## 与 Issues 的区别

### 使用 Discussions 的场景

- 💬 一般性问题和讨论
- 💡 功能想法和建议
- ❓ 使用帮助和指导
- 🎉 展示和分享

### 使用 Issues 的场景

- 🐛 明确的 Bug 报告
- ✨ 具体的功能请求
- 📝 文档改进建议
- 🔧 技术债务和改进

**简单来说**:
- **Discussions**: 用于讨论和交流
- **Issues**: 用于跟踪和解决问题

## 搜索和筛选

### 搜索讨论

使用 Discussions 页面的搜索框可以：

- 搜索标题和内容
- 按分类筛选
- 按作者筛选
- 按标签筛选

### 常用搜索

- `label:question` - 查找所有问题
- `category:ideas` - 查找所有想法
- `author:username` - 查找特定用户的讨论

## 最佳实践

### 对于提问者

1. **先搜索**: 搜索是否已有相关讨论
2. **提供上下文**: 包括环境、版本、操作步骤
3. **使用代码块**: 格式化代码和日志
4. **及时反馈**: 回复后续问题和解决方案

### 对于回答者

1. **友善耐心**: 保持友好和专业
2. **提供示例**: 如果可能，提供代码示例
3. **引用文档**: 指向相关文档链接
4. **标记答案**: 如果问题已解决，可以帮助标记

### 对于维护者

1. **及时响应**: 尽量及时回复讨论
2. **分类管理**: 确保讨论在正确的分类中
3. **标记重要讨论**: 使用 Pinned 功能标记重要讨论
4. **从讨论到 Issue**: 如果讨论明确了问题，可以创建 Issue 跟踪

## 社区准则

参与 Discussions 时，请遵守以下准则：

1. **尊重他人**: 保持友好和尊重
2. **保持相关**: 讨论应与 OpenTruss 项目相关
3. **提供价值**: 分享有用的信息和经验
4. **遵守规范**: 遵守 GitHub 社区准则

## 获取帮助

如果在使用 Discussions 时遇到问题：

1. 查看 [GitHub Discussions 文档](https://docs.github.com/en/discussions)
2. 联系项目维护者
3. 在 Discussions 中提出使用问题
